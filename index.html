HTML

  • The head element contains information about the webpage.
  • < !DOCTYPE html> This declaration tells the browser what document type it should expect when it is opened.
  • < html lang="en"> This represents the root, or start, of our HTML document. It also indicates the language to the browser
  • < title> Defines the Title of the Webpage important for Search Engine Optimization (SEO).
  • < style> Defines the CSS styles associated with a website.
  • < base> Defines the base url for a webpage.
  • < link> Connects an external resource to the HTML document.
  • < meta> Defines metadata such as the character set, description, keywords, author, and viewport. More about this here.
  • < script> Embeds client-side scripts in the HTMl document.
  • The body element represents the visible content shown to the user.
  • < section> allows the content to be divded into seperate sections for easier access to certain information.
  • < div> Used to create a division in the page.
  • < span> Used to create an inline division.
  • < h1-6> Used to display a heading on the webpage. 1-6 Display different sized headings.
  • < p> Represents a paragraph or a block of text.
  • < ul> Unordered List that creates a bulleted list.
  • < ol> Ordered List that creates a numerical list.
  • < li> allows you to list the items in that list seperately.
  • < img> Contains information about images that are displayed on the webpage. Can contain various attributes; the src attribute is the most important because it defines the location of the image, alongside the alt attribute that allows you describe the image, and the width and height elements that allow you to adjust the picture size.
  • < br> Creates a line of empty space, or a line break.
  • < a> Stands for anchor and creates links to the same webpage or other webpages. href allows you to decide where the redirect eoll be sent.

Git

  • ls List the files and directories.
  • cd allows us to change directories.
  • pwd is short for "print working directory," will print out the directory that we are currently in to the terminal.
  • mkdir directory name Allows you to create a new directory and name it.
  • touch File name and .type Allows you to create a file and name it within the current directory
  • ls -al ~/.ssh Allows you to see any existing ssh keys.
  • ssh-keygen -t ed25519 -C “your_email@example.com” Used to create an ssh key
  • eval “$(ssh-agent -s)” Starts the ssh agent.
  • git revert sha key reverts to a previous form using the sha key as the refrence to revert back to
  • Git Command Cheat Sheet
  • code . Used to open vs code
  • git status: checks what branch we are currently on
  • git checkout -b branch-name:creates a new branch and switches to it
  • git add -A adds modifications in the current working branch to the staging area. The -A flag indicates that we want to add all changes.
  • git commit -m "notes" Commits the changes and adds a note as to what the commit contains.
  • git pull origin main pulls the most upto date code from github to ensure you are working on the most upto date code.
  • git push origin branch that needs to be implemented name Pushes the changes from our local drive to the remote github Repo to update the Main Code.
